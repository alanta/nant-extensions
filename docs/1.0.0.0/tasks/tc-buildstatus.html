<tc-buildstatus>

[This is preliminary documentation and subject to change.]

Reports the build status to TeamCity.

Parameters

Attribute Type Description Required
message string The message to include with the report status. False
type StatusType The status type to report to TeamCity. The default value is Normal False
failonerror bool False
force bool Force task execution regardless of if this is a TeamCity build. The default value is false. False
if bool False
unless bool False
verbose bool False

Examples

  • Sets the build status to "failed" and replaces the current message. 
    <tc-buildstatus message="The build has failed"
                type="Error" />
  • Appends the code coverage value to the current build status message. {build.status.text} is an optional substitution pattern which represents the status, calculated automatically by TeamCity using passed test count, compilation messages and so on. 
    <tc-buildstatus message="{build.status.text}, Code coverage: ${math::round(double::parse(codecoverage))}%"/>
